## 1. 프로젝트 개요

### 1.1 프로젝트 명

**AquaWorld (가칭)** — 정식 명칭은 베타 출시 전 사용자 테스트를 통해 확정

### 1.2 한 줄 소개

> 「**내 손안의 살아있는 3D 수족관** — 물고기를 키우고, 꾸미고, 친구와 공유하는 힐링 게임」

### 1.3 프로젝트 비전

바쁜 일상 속에서 짧은 시간이라도 마음의 평화를 찾을 수 있는 디지털 안식처를 제공한다. 사용자는 자신만의 수조를 가꾸며 작은 생명을 돌보는 경험을 통해 정서적 만족감을 얻고, 친구와의 가벼운 소셜 인터랙션을 통해 관계의 따뜻함을 유지한다.

### 1.4 프로젝트 목표

- **정성적 목표:** 사용자가 하루에 평균 3회 이상 자발적으로 앱을 열어 자신의 수조를 들여다보는 일상의 한 부분이 되는 앱
- **정량적 목표:** 출시 6개월 내 누적 방문자 100만, MAU 15만, D7 리텐션 25% 달성
- **수익 목표:** 출시 12개월 내 월 매출 1억 원 (ARPU $1.5 기준)

### 1.5 핵심 가치 제안

기존의 수족관 시뮬레이션 게임이 평면적이고 단순한 터치 인터랙션에 머무는 반면, 본 앱은 다음 세 가지 가치를 동시에 제공한다.

- **진짜 같은 3D 경험:** Three.js 기반의 완전한 3D 렌더링으로 수조를 모든 각도에서 감상 가능
- **힐링과 컬렉션의 조화:** 스트레스 없는 사육 시스템 + 수집 욕구를 자극하는 희귀종 시스템
- **가벼운 소셜 연결:** 부담 없는 친구 방문과 공유 기능으로 관계의 따뜻함 유지

## 2. 시장 분석 및 기회

### 2.1 글로벌 모바일 게임 시장 동향

2025년 기준 글로벌 모바일 게임 시장은 약 920억 달러 규모로, 그중 캐주얼 게임 카테고리는 전체의 약 27%를 차지한다. 특히 코로나19 이후 '힐링 게임' 또는 '코지 게임(Cozy Game)' 장르가 폭발적으로 성장했으며, *Animal Crossing: Pocket Camp*, *AbyssRium*, *Tap Tap Fish* 등이 대표 사례로 꼽힌다.

### 2.2 힐링/사육 시뮬레이션 게임 시장

힐링 시뮬레이션 장르는 다음과 같은 특징으로 다른 게임 장르 대비 차별화된 강점을 가진다.

- **높은 리텐션:** 평균 D30 리텐션이 일반 캐주얼 게임 대비 1.8배 높음
- **긴 LTV:** 충성 사용자의 평생 가치가 일반 게임 대비 약 2.3배
- **폭넓은 사용자층:** 10대부터 50대까지 고른 분포, 특히 여성 사용자 비율 60% 이상
- **낮은 이탈률:** 경쟁 요소가 적어 스트레스로 인한 이탈이 적음

### 2.3 PWA의 추가 강점

네이티브 앱 대비 PWA가 제공하는 차별화된 이점.

- **앱스토어 없이 즉시 접근:** URL 공유만으로 유입, 설치 마찰 제거
- **크로스 디바이스:** iOS, Android, PC 모두 동일한 코드베이스로 지원
- **바이럴 확산 용이:** 링크로 공유 가능, SNS에서 즉시 실행
- **업데이트 즉시 반영:** 서비스워커 캐시 무효화만으로 전체 사용자에게 동시 배포

### 2.4 경쟁 분석

#### 주요 경쟁 앱

| 앱 이름 | 강점 | 약점 | 우리의 차별점 |
|---|---|---|---|
| **AbyssRium** | 귀여운 아트, 단순한 조작, 전 세계 7천만 다운로드 | 2.5D 고정 시점, 인터랙션 단조로움, 꾸미기 자유도 낮음 | 완전한 3D 자유 시점, 사용자 커스터마이징 강화 |
| **Tap Tap Fish** | 종 다양성, 도감 시스템 | 탭 중심의 단순 게임플레이, 시각적 다양성 부족 | 3D 공간 꾸미기, 다양한 환경(심해/강/판타지) |
| **Fish Tycoon** | 교배/유전 시스템 | 올드한 UI, 모바일 최적화 부족, 사회적 기능 없음 | 모던한 UI/UX, 소셜 기능 통합 |
| **Alba: A Wildlife Adv.** | 힐링 컨셉, 자연 친화 | 유료 앱, 일회성 콘텐츠 | 무료 진입 + 지속적 콘텐츠 업데이트 |

#### 시장 빈 공간

경쟁 분석 결과 다음과 같은 시장의 빈 공간(market gap)을 확인했다.

1. '완전한 3D + 자유 시점'을 제공하는 모바일 수족관 게임 부재
2. 한국 사용자 정서(감성적 UI, 한국적 미감)에 최적화된 힐링 게임 부족
3. '개인 사육 + 가벼운 소셜'의 균형을 잘 잡은 앱 부족 (대부분 단독 플레이만 가능)
4. ASMR 사운드 + 백그라운드 재생을 적극 활용한 앱 부재
5. **앱스토어 없이 즉시 플레이** 가능한 고품질 수조 게임 부재

## 3. 타겟 사용자 정의

### 3.1 핵심 타겟 사용자 (Primary)

> **20-35세 여성, 직장인 또는 대학생**
> 도시 거주, 1인 가구 비율 높음, 반려동물 키우기 어려운 환경
> 스트레스 해소와 감성적 만족을 위한 콘텐츠를 적극 소비

### 3.2 페르소나

#### 페르소나 1: 직장인 김지은 (28세)

- **배경:** 서울 거주, 마케팅 회사 3년차, 1인 가구, 반려동물 키우고 싶지만 출장이 잦아 포기
- **니즈:** 퇴근 후 짧게라도 마음의 평화를 찾을 수 있는 활동, SNS에 공유할 수 있는 예쁜 콘텐츠
- **앱 사용 시나리오:** 출근길 지하철에서 5분, 점심시간 후 3분, 자기 전 침대에서 10분 정도 자신의 수조를 들여다보며 휴식
- **결제 의향:** 월 5,000~15,000원 정도, 특히 시즌 한정 아이템에 적극적

#### 페르소나 2: 대학생 박서연 (22세)

- **배경:** 대학교 3학년, 자취 시작 6개월차, 인스타그램 활성 이용자
- **니즈:** 친구들과 공유할 수 있는 귀여운 콘텐츠, 스트레스 없는 가벼운 게임
- **앱 사용 시나리오:** 수업 사이 짬짬이, 친구와 통화하며 수조 구경, 예쁜 수조 스크린샷을 인스타 스토리에 공유
- **결제 의향:** 월 0~5,000원, 광고 시청 의향 높음, 친구 추천으로 보상 받기 선호

### 3.3 보조 타겟 사용자 (Secondary)

- **30-50대 직장인 남성:** 어린 시절 키웠던 수족관에 대한 향수, 진짜 수조 관리의 번거로움을 디지털로 대체
- **초등학생 자녀를 둔 부모:** 자녀와 함께 즐길 수 있는 안전한 콘텐츠
- **ASMR/명상 콘텐츠 애호가:** 백그라운드로 켜놓고 사운드를 즐기는 사용자층

## 4. 핵심 컨셉 및 차별화 전략

### 4.1 핵심 컨셉

본 앱은 세 가지 축을 중심으로 설계된다.

> **① 진정성 있는 3D 경험 (Authentic 3D Experience)**
> Three.js로 구현된 자유로운 카메라 워크, 실시간 물리 시뮬레이션, 빛의 굴절과 물결 효과

> **② 잔잔한 게임플레이 (Calm Gameplay)**
> 강제 미션 없음, 죽지 않는 물고기, 부담 없는 진행 속도, 언제든 떠나도 괜찮은 게임

> **③ 가벼운 소셜 (Light Social)**
> 친구 수조 방문, 비동기 인터랙션, 경쟁 없는 협력형 소셜 디자인

### 4.2 핵심 차별화 포인트

#### ① 기술적 차별화: 완전한 3D 렌더링

경쟁 앱들은 대부분 2.5D 또는 고정 시점의 3D를 사용한다. 본 앱은 다음과 같은 기술적 차별화를 추구한다.

- **자유 카메라:** 360도 회전, 줌인/줌아웃, 위에서 내려다보기 시점 등 자유로운 시점 변경
- **물고기 시점 모드:** 물고기 한 마리를 선택해 그 시점에서 수조 내부를 헤엄치는 체험
- **실시간 셰이더 효과:** 물결, 빛의 굴절, 거품, 수초의 흔들림을 실시간 렌더링
- **물리 시뮬레이션:** 물고기의 자연스러운 움직임, 충돌, 무리 행동(boids algorithm)

#### ② 플랫폼 차별화: PWA로 진입 장벽 제거

- **설치 없이 즉시 실행:** URL 또는 QR 코드 하나로 게임 시작
- **홈 화면 추가:** iOS/Android 모두 홈 화면에 앱처럼 설치 가능 (standalone 모드)
- **오프라인 지원:** 서비스워커로 핵심 에셋 캐싱, 네트워크 없이도 수조 감상 가능

#### ③ 감성적 차별화: 한국 정서 반영

- 국악/뉴에이지 풍의 BGM 옵션 (서양 명상 음악과 차별화)
- 한국 토종 어종 컬렉션 (쏘가리, 버들치, 각시붕어 등 — 교육적 가치도 함께)
- 사군자, 한지 등 한국적 미감의 꾸미기 아이템

### 5.2 MVP 핵심 기능 상세

#### 5.2.1 기본 수조 시스템

- **초기 수조 제공:** 신규 가입 시 기본 사이즈의 수조 1개 자동 지급 (가로 60cm 기준 가상 사이즈)
- **배경 환경 선택:** MVP에서는 5종 환경 제공 (산호초, 심해, 한국 강, 아마존, 우주)
- **수조 확장:** 최대 5개까지 슬롯 확장 가능, 추가 슬롯은 인앱 결제 또는 광고 시청
- **수조 수용량(마릿수 상한):** 수조 한 개에 배치할 수 있는 물고기 수에 상한을 둔다. 기본 8마리에서 시작하며, 수조 레벨업/확장으로 단계적으로 증가(8 → 12 → 16 → 20). 상한에 도달하면 새 물고기를 배치하기 전에 기존 물고기를 보관함으로 빼야 한다 (→ 5.2.7 참조)
- **실시간 조명:** 낮/밤 자동 전환, 사용자가 수동으로 조명 모드 선택도 가능

#### 5.2.2 물고기 사육 시스템

- **먹이주기:** 하루 최대 3회, 화면 터치로 먹이를 뿌리는 인터랙션, 먹이 종류에 따라 성장 속도/색깔 변화
- **자동 성장:** 실시간으로 자라며 알→치어→성어→대형어 단계로 진화
- **죽지 않음:** 스트레스 방지를 위해 물고기는 죽지 않으며 '아픔' 상태도 없음. 단, 먹이를 안 주면 자라지 않을 뿐
- **기분 시스템:** 쾌적도(수조 청결도, 다른 물고기와의 궁합)에 따라 '행복', '평범', '심심함' 표정 변화

#### 5.2.3 물고기 가챠 시스템

- **알(Egg) 형태로 획득:** 기본 알, 희귀 알, 전설 알의 3티어
- **부화 시간:** 기본 알 5분, 희귀 알 30분, 전설 알 2시간 (광고 시청으로 단축 가능)
- **MVP 보유 종:** 10종 (커먼 5종, 레어 3종, 에픽 1종, 레전더리 1종)
- **획득 경로:** 일일 로그인 보상, 먹이주기 누적 보상, 인앱 결제, 광고 시청

#### 5.2.4 꾸미기 시스템

- **배치 가능 오브젝트:** 수초 8종, 바위 6종, 유목 4종, 장식품 8종 (MVP 기준)
- **3D 드래그 배치:** 사용자가 직접 위치와 회전을 자유롭게 조정
- **저장 슬롯:** 수조별 꾸미기 프리셋 3개 저장 가능

#### 5.2.5 도감 시스템

- **종 도감:** 획득한 모든 물고기 종 기록, 잠금 해제 시 실루엣→풀 이미지
- **정보 제공:** 실제 어종의 경우 학명, 서식지, 특징 등 교육적 정보 함께 제공
- **진척도:** 전체 도감 완성도 % 표시로 컬렉션 욕구 자극

#### 5.2.6 스크린샷 및 공유

- **포토 모드:** UI를 모두 숨기고 자유롭게 각도를 잡아 사진 촬영
- **필터/프레임:** 5종 필터, 3종 프레임 제공
- **공유:** Web Share API를 통해 인스타그램, 카카오톡 등 외부 SNS 연동
- **워터마크:** 공유 시 작은 앱 로고 자동 삽입 (바이럴 효과)

#### 5.2.7 물고기 보관함 & 배치 시스템

물고기는 "보관함(소유)"과 "수조(전시)"의 두 상태를 가진다. 가챠로 획득한 물고기가 무조건 현재 수조에 쌓이지 않고, 사용자가 어떤 물고기를 어느 수조에 둘지 직접 큐레이션할 수 있다. (꾸미기 시스템의 인벤토리 패턴을 물고기에 동일하게 적용)

- **보관함(아쿠아 박스):** 부화/획득한 모든 물고기가 먼저 보관함에 들어간다. 보관함 용량에는 상한이 없으며, 종/희귀도/획득일 기준 정렬·필터 제공
- **수조에 배치:** 보관함에서 물고기를 선택해 현재 수조에 넣는다. 수조의 **마릿수 상한**(5.2.1)을 초과해 배치할 수 없다
- **수조에서 빼기:** 수조의 물고기를 다시 보관함으로 회수한다. 물고기는 **죽거나 사라지지 않으며**(힐링 철학 유지) 성장 단계·기분 등 상태는 보존된다
- **수조 간 이동:** 멀티 수조(최대 5개) 환경에서 같은 물고기를 다른 수조로 옮길 수 있다 (보관함 경유)
- **테마 큐레이션:** 환경(산호초/심해/강/아마존/우주)에 어울리는 물고기만 선별 배치해 자신만의 수조 컨셉을 연출
- **혼잡 방지 효과:** 상한 + 보관함 구조로 한 수조에 물고기가 과도하게 몰리는 혼잡을 자연스럽게 방지하고, 3D 렌더링 부하도 배치된 물고기 수로 제한된다
- **3D 씬 동기화:** 수조에 배치된 물고기만 렌더링 대상이며, 보관함의 물고기는 도감(보유 기준)에는 기록되지만 씬에는 등장하지 않는다

#### 5.2.8 번식(짝짓기) 시스템

키운 물고기로 새 물고기를 얻는 **자가 순환 컬렉션 루프**. 알을 상점에서 사는 것 외에, 애착이 붙은 내 물고기를 짝지어 다음 세대를 얻는 재미를 더한다. 가챠 시스템(5.2.3)과 별개의 알 획득 경로로, 기존 인큐베이터·부화 파이프라인을 그대로 재사용한다.

- **자격:** **같은 종**의 **성어 이상**(성어/대형어) 물고기 2마리. 서로 다른 개체여야 한다
- **비용·제약:** 짝짓기 1회당 **Pearl 200** 소비. 사용한 두 부모는 각각 **12시간 재짝짓기 쿨다운**에 들어간다. **부모는 사라지지 않으며**(힐링 철학 유지) 상태도 그대로 유지된다
- **결과물:** 부모 등급에 맞는 **번식 알**이 인큐베이터에 담긴다(커먼→기본 알, 레어→희귀 알, 에픽/레전더리→전설 알). 이후 부화 흐름은 일반 알과 동일
- **부화 규칙:** 기본적으로 **부모와 같은 종**이 나오며, **낮은 확률(8%)로 한 단계 상위 등급** 풀의 종이 나온다("대박"). 예) 커먼 부모 → 대부분 같은 커먼 종, 드물게 레어 종. 이 상위 등급 경로가 커먼만으로도 상위 종을 노릴 수 있는 완만한 성장 사다리가 된다
- **서버 권위:** 클라우드 유저는 Pearl 차감·쿨다운·알 지급을 서버(`breedFish`)가 권위로 처리(낙관적 UI + 실패 시 롤백). 종 추첨은 부화 시점에 서버에서 수행. 게스트는 로컬에서 동일 규칙 적용
- **진입점:** 메인 수조 화면 좌측 `💞 짝짓기` 패널. 성어 이상 후보가 2마리 이상일 때만 노출되며, 부모 2슬롯을 채우고 짝짓기를 실행한다
- **향후 확장(비MVP):** 변이 색상(반짝이 개체), 부모 능력치 유전, 특정 조합 한정 종 등으로 수집 깊이를 더 넓힐 수 있다

## 8. 기술 스택 및 아키텍처

### 8.1 기술 스택 선정

| 영역 | 기술/라이브러리 | 선정 이유 |
|---|---|---|
| **앱 프레임워크** | React 18 + Vite 6 | 빠른 빌드, 풍부한 생태계, PWA 최적화 |
| **배포 형태** | PWA (Progressive Web App) | 앱스토어 없이 설치 가능, 크로스 디바이스 |
| **서비스워커** | Workbox (vite-plugin-pwa) | 오프라인 캐싱, 백그라운드 동기화 |
| **3D 렌더링** | Three.js (pure WebGLRenderer) | 네이티브 WebGL 직접 사용, 최고의 성능 |
| **라우팅** | React Router v6 (HashRouter) | SPA 라우팅, PWA 배포 호환 |
| **상태 관리** | Zustand v5 | 가볍고 직관적, TypeScript 친화적 |
| **언어** | TypeScript (strict) | 타입 안정성, 개발 생산성 |
| **백엔드** | Firebase v11 | BaaS로 초기 개발 속도 확보, 실시간 DB |
| **인증** | Firebase Auth + 소셜 SDK | Google, 카카오 로그인 통합 |
| **결제** | Web Payments API + 서버 검증 | 크로스 플랫폼 결제 |
| **광고** | Google AdSense / AdMob Web | 웹 기반 광고 |
| **분석** | Firebase Analytics | 사용자 행동 분석, 코호트 분석 |
| **크래시 리포팅** | Sentry | 실시간 오류 추적, 성능 모니터링 |
| **푸시 알림** | Web Push API + FCM | 브라우저 푸시 알림 |

### 8.2 아키텍처 개요

앱은 다음과 같은 3-Tier 아키텍처를 따른다.

```
[Client Layer]   React + Three.js (WebGLRenderer on <canvas>)
                 Service Worker (Workbox) — 오프라인 캐시
       ↕  (HTTPS API / Realtime WebSocket)
[BaaS Layer]     Firebase (Auth, Firestore, Functions, Storage)
       ↕
[3rd Party]      AdSense/AdMob Web, AppsFlyer, Sentry, Web Payments
```

### 8.3 PWA 구성

- **manifest.json**: 앱 이름, 아이콘(192×192, 512×512), `display: standalone`, `theme_color`
- **서비스워커**: Workbox `StaleWhileRevalidate` 전략으로 JS/CSS/이미지 캐싱
- **iOS 지원**: `apple-mobile-web-app-capable` 메타 태그, iOS 아이콘 링크
- **설치 프롬프트**: `beforeinstallprompt` 이벤트 캡처 후 적절한 시점에 표시

### 8.4 핵심 기술 과제

#### ① 3D 렌더링 성능 최적화

모바일 브라우저에서 Three.js를 안정적으로 구동하기 위한 최적화 전략.

- **폴리곤 수 제한:** 물고기 한 마리당 500~1500 polygons
- **LOD(Level of Detail) 적용:** 거리에 따라 모델 디테일 조정
- **Texture Atlas:** 여러 텍스처를 하나의 이미지로 결합하여 draw call 감소
- **Instanced Mesh:** 동일한 객체(예: 수초)는 인스턴싱으로 처리
- **Frame rate 동적 조정:** 저사양 기기에서 30fps, 고사양에서 60fps
- **ResizeObserver**: 캔버스 크기 반응형 조정, 불필요한 재렌더링 방지

#### ② 배터리/성능 효율

- Page Visibility API로 탭 비활성 시 RAF(requestAnimationFrame) 일시 정지
- ASMR 모드: 화면 어둡게 + 사운드만 재생하는 저전력 모드 제공
- 물리 시뮬레이션 단순화 옵션 (배터리 절약 모드)

#### ③ 오프라인 대응

- 서비스워커가 정적 에셋(JS, CSS, 이미지, 3D 모델) 캐싱
- Zustand 상태를 localStorage에 persist하여 오프라인에서도 수조 상태 유지
- 온라인 복귀 시 Firebase와 자동 동기화

- **틱톡 시드 콘텐츠:** 30초 분량 ASMR 수조 영상, 부화 과정 영상 등으로 사전 인지도 확보
- **베타 테스터 모집:** URL 공유만으로 100~500명 비공개 베타 가능 (앱스토어 불필요)

### 10.2 출시 직후 (Launch) 전략

- **PWA 설치 유도:** "홈 화면에 추가" 인앱 가이드, 설치 시 보너스 Pearl 지급

- **바이럴 메커니즘:** 예쁜 수조 사진 → 자동 워터마크 → Web Share API로 SNS 공유 → 자연 확산
- **친구 초대 보상:** 초대 링크(URL) 공유 — 친구가 가입하면 양쪽 모두 희귀 알 1개 지급
- **시즌 이벤트:** 설날, 발렌타인데이, 벚꽃 시즌, 여름, 추석, 크리스마스 등 주기적 이벤트로 재방문 유도
- **커뮤니티 운영:** 공식 디스코드/카카오톡 오픈채팅, 사용자가 자기 수조 자랑할 수 있는 공간

## 부록 A. 용어 정리

| 용어 | 정의 |
|---|---|
| **MVP** | Minimum Viable Product. 최소 기능을 갖춘 초기 제품 |
| **PWA** | Progressive Web App. 웹 기술로 만든 앱스토어 없이 설치 가능한 앱 |
| **서비스워커** | 브라우저 백그라운드에서 실행되는 스크립트로 오프라인 캐싱, 푸시 알림 등을 담당 |
| **DAU/MAU** | Daily/Monthly Active Users. 일일/월간 활성 사용자 수 |
| **ARPU** | Average Revenue Per User. 사용자당 평균 매출 |
| **리텐션** | Retention. 사용자가 일정 기간 후에도 다시 앱을 사용하는 비율 |
| **ASO** | App Store Optimization. 앱스토어 검색 최적화 (PWA는 SEO로 대체) |
| **SEO** | Search Engine Optimization. 검색 엔진 최적화 |
| **WebXR** | Web 브라우저에서 AR/VR을 구현하는 W3C 표준 API |
| **LOD** | Level of Detail. 거리에 따라 3D 모델 디테일을 조정하는 기법 |
| **Boids Algorithm** | 물고기 떼나 새 떼의 자연스러운 무리 행동을 시뮬레이션하는 알고리즘 |
| **RAF** | requestAnimationFrame. 브라우저 렌더링 루프에 맞춰 애니메이션을 실행하는 Web API |
| **CPI** | Cost Per Install. 앱 설치 1건당 광고 비용 (PWA는 CPV/CPA로 측정) |
| **LTV** | Lifetime Value. 사용자 한 명이 평생에 걸쳐 발생시키는 가치 |
| **BaaS** | Backend as a Service. 백엔드 기능을 클라우드 서비스로 제공받는 형태 |
| **PoC** | Proof of Concept. 개념 증명, 기술 검증 |
